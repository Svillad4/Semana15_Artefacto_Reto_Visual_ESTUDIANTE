# Guía de recursos visuales

Un recurso visual no es decoración. Debe ayudar a entender mejor el resultado.

## Qué recurso elegir

| Necesidad | Recurso recomendado |
|---|---|
| Comparar producción por producto | Gráfico de barras |
| Mostrar números exactos | Tabla resumen |
| Explicar el proceso del programa | Diagrama de flujo |
| Evidenciar que el código funcionó | Captura de terminal |
| Resumir problema, solución y resultado | Infografía o diapositiva |

## Requisitos del recurso visual

El gráfico o tabla debe tener:

- título claro;
- etiquetas comprensibles;
- datos relacionados con el problema;
- interpretación breve;
- coherencia con el reporte generado.

## Preguntas para interpretar el gráfico

1. ¿Qué producto tuvo mayor producción?
2. ¿Qué diferencia se observa entre los productos?
3. ¿Qué decisión podría tomar la comunidad con esa información?
4. ¿El gráfico permite entender el resultado más rápido que solo texto?
